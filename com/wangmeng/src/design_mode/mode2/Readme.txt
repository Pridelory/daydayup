观察者模式
气象站示例
example1是自己实现
example2是用java内置的观察者模式来实现
其中用java内置实现时，主题是继承Observable，如果此时有其他继承，那么这个就不适合用内置了
而且在使用java内置实现时，还可以实现观察者向被观察者取数据的功能。也可以自己实现
最好自己实现观察者模式，因为简单而且灵活性又高

思路：主题中和观察者之间是松耦合的。主题内部维护了一个观察者接口的列表，当增加或者减少对改主题的观察者
时，主题结构都不会改变。主题不关注观察者的内部结构，只要其观察者实现了update方法即可。这样当主题信息改变
时候，会遍历观察者，并统一调用update方法挨个通知观察者。对于观察者，需要维持一个主题对象的引用，在初始化
时把相应的主题对象传入进来即可。

观察者可以主动向主题注册，也可主动向主题删除，就跟定报纸一样，决定权在订阅者手里。
